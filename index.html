Bienvenue dans la documentation de mon infrastructure. Ce projet démontre comment automatiser la création d'un artefact immuable (Docker) et son orchestration sur une plateforme Kubernetes moderne.

2. Processus Technique

A. Build Immuable (Packer)

Plutôt que de configurer un serveur à la main, nous utilisons Packer pour créer une image Docker "prête à l'emploi". On injecte notre index.html directement dans le moule de l'image Nginx.

packer build image.pkr.hcl

B. Orchestration (Ansible)

Ansible agit ici comme le pilote. Il communique avec l'API Kubernetes pour déclarer l'état souhaité :

  • Deployment : Assure que 2 réplicas de notre application tournent en permanence.
  • Service : Expose l'application via un NodePort pour la rendre accessible.

3. Défis Techniques & Solutions

Problème rencontré : Lors de l'exécution d'Ansible, le système Ubuntu bloquait l'installation de la librairie kubernetes (PEP 668).
Solution : Utilisation du flag --break-system-packages pour forcer l'installation dans l'environnement GitHub Codespaces, permettant à Ansible d'utiliser l'interprète Python système.

4. Guide d'utilisation rapide

Pour reproduire ce déploiement, utilisez le Makefile inclus :

# 1. Préparer l'environnement et le cluster
make setup

# 2. Builder et importer l'image
make build

# 3. Déployer sur Kubernetes
make deploy
