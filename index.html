Concept de l'Atelier

Cet atelier simule une chaîne de production réelle. L'enjeu est de passer d'un code source (HTML) à un service haute disponibilité sur Kubernetes sans aucune configuration manuelle "ad-hoc".

Nous utilisons la philosophie de l'Infrastructure as Code (IaC) et de l'Immutabilité : on ne modifie jamais un serveur en production, on reconstruit une nouvelle image et on la déploie.

1. Build Artifact

Packer crée l'image Docker

2. Orchestration

Ansible gère les objets K8s

3. Execution

Cluster K3d Haute Dispo

Automatisation des Dépendances

Le plus gros défi est la gestion de l'environnement GitHub Codespaces (Ubuntu 24.04). Nous automatisons l'installation via le Makefile :

  • Nettoyage des sources APT (Yarn error fix).
  • Installation forcée de kubernetes-python via --break-system-packages pour lier Ansible à l'OS.
  • Installation des binaires Packer et K3d.

Le Pipeline technique

# Build Packer
# Injecte index.html -> Nginx
packer build image.pkr.hcl

# Injection K3d
# Transfert l'image vers les nodes
k3d image import custom-nginx -c lab

# Run Ansible
# Applique le Deployment & Service
ansible-playbook deploy.yml

Automatisation Totale : Le "Master Script"

Pour ne plus jamais taper les commandes une par une, nous avons centralisé l'intelligence dans un script de déploiement automatique.

# Pour tout lancer d'un coup :
chmod +x deploy.sh && ./deploy.sh
